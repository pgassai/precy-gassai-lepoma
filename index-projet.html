Présentation du projet


Dans le cadre d’un projet universitaire SAE (Situation d’apprentissage et d’évaluation), notre équipe de quatre membres avait pour mission de créer une base de données en accord avec un cahier des charges détaillant les règles métier, les besoins et contraintes, ainsi que les jeux de données à intégrer. Notre objectif principal était de maîtriser et appliquer les différentes étapes nécessaires à la mise en place d'une base de données relationnelle.



Démarche


Pour mener à bien ce projet, nous avons débuté par un processus de traitement des données, comprenant le nettoyage afin d'éliminer les données incohérentes et de mieux appréhender la structure des données pour guider notre réflexion quant à la modélisation des données, en vue de leur exploitation. Ensuite, nous avons élaboré un modèle conceptuel des données (MCD) suivi d'un modèle logique des données (MLD). Ces modèles nous ont permis de concevoir et d'implémenter la base de données dans un SGBD spécifique. Enfin, nous avons réalisé et documenté un audit détaillé pour assurer la qualité, la cohérence et l'intégrité des données.



Outils et/ou langage utilisé

Access, Excel

Présentation du projet


Dans le cadre du projet universitaire SAE "Description et prévision de données temporelles", notre équipe se focalise sur l'analyse et la prévision de séries chronologiques de la production d'électricité aux États-Unis sur 22 ans, de 2001 à 2022. Ces données, provenant de l'Administration de l'information sur l'énergie (EIA), couvrent différents types de production électrique, notamment le charbon, le pétrole, le gaz naturel, les énergies renouvelables, etc. Notre objectif est d'analyser les données mensuelles de la production d'électricité provenant des autres gaz en kw/H afin de fournir une prévision pour l’année suivante 2023.



Démarche


Nous débutons par une analyse des tendances de la série temporelle à l'aide de filtres de moyennes mobiles et de courbes de régression des moyennes annuelles. Ensuite, nous calculons et représentons graphiquement les coefficients saisonniers, procédons à la décomposition de la série, et analysons la série corrigée des variations saisonnières, en commentant les résultats et identifiant les valeurs mal ajustées par le modèle en étudiant les résidus.
Pour prévoir la production d'électricité pour 2023, nous utilisons trois méthodes différentes : la méthode Alternative, la méthode Holt-Winters et la méthode ARMA. De plus, nous effectuons une rétro-prévision pour 2022 en utilisant les données de 2001 à 2021. Nous évaluons la performance et la précision de ces méthodes en calculant l'erreur quadratique moyenne pour chacune d'entre elles.
Les résultats nous ont permis d’identifier la méthode Alternative comme la plus adéquate dans ce cas précis.



Outils et/ou langage utilisé


Rstudio (library tseries, forecast)

Présentation du projet


Dans le cadre du projet universitaire SAÉ "Collecte automatisée de données web", notre équipe se focalise sur l'automatisation de la collecte et de l'intégration de données provenant du web dans un entrepôt de données. Cette mission exige une compréhension approfondie des données web ainsi que l'utilisation d'interfaces de programmation d'application (API) pour récupérer les données en ligne.



Démarche


Notre démarche débute par la modélisation et la création de la structure de l'entrepôt de données. Ensuite, nous procédons à l'intégration des données fournies, notamment celles de la base StockEtablissement SIRENE, puis nous développons un programme en Python pour les manipuler (nettoyage, prétraitement, jointures, etc.) en vue de les stocker dans l'entrepôt de données préalablement créé.
Nous avons également eu recours à l'API Adresse pour convertir une adresse en coordonnées géographiques. Une requête HTTP est envoyée à l'API avec une adresse spécifique préalablement traitée, et les résultats sont obtenus au format JSON, puis intégrés dans notre base de données.
Enfin, nous explorons le web-scraping avec Python pour extraire des informations à partir de Google Maps en utilisant les bibliothèques Selenium et BeautifulSoup. Le contenu HTML de la page est extrait, nous permettant ainsi de récupérer l'adresse e-mail de l'établissement si celle-ci est renseignée. Ces informations sont ensuite stockées et enrichissent notre base de données.



Outils et/ou langage utilisé

Python ( bibliothèque Pandas, Requests, Selenium, BeautifaulSoup), Chromedriver, Google Maps, API Adresse (api-adresse.data.gouv.fr)

Présentation du projet


Pour mon projet personnel, j'ai entrepris de créer un système de classification d'images pour reconnaître des dessins. Mon objectif était de comprendre le processus de classification d'images afin de développer un jeu similaire à "Quick, Draw!" en utilisant TensorFlow.
Quick, Draw!" est un jeu en ligne qui met au défi les joueurs de dessiner un croquis, puis une intelligence artificielle tente de deviner ce que représentent les dessins.



Démarche


Ma démarche a débuté par le développement d'un code Python utilisant Flask pour construire une application web. Ensuite, j'ai intégré un modèle de réseau de neurones convolutifs (CNN) pré-entraîné, InceptionV3, pour la classification d'images. J'ai également développé des fonctions pour prétraiter les images et prédire le dessin soumis par l'utilisateur.
Par la suite, j'ai créé un code HTML pour définir un canvas où l'utilisateur peut dessiner. Ce canvas est accompagné d'un bouton permettant de télécharger le dessin et d'un paragraphe affichant le résultat de la prédiction.



Outils et/ou langage utilisé

Python ( bibliothèque TensorFlow, ,TensorFlow.js Flask, PIL, NumPy), Navigateur Web, HTML
